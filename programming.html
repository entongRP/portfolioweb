Travel Packing List Web
Inspiration & Thought Process

I was inspired to create this app after realizing how stressful and chaotic packing for trips could be. I often forgot essential items or struggled to remember quantities, which led me to think about creating a simple digital tool to manage packing. My thought process started with identifying the core problem: travelers needed an organized way to track items, mark them as packed, and see everything at a glance. During development, I faced challenges managing the dynamic state of items and quantities, ensuring real-time UI updates, and handling edge cases. This project taught me the importance of thoughtful design, clear logic, and rigorous testing to provide a smooth user experience.

Assignment List Tracker Web
Inspiration & Thought Process

I created this app to solve a personal problem of keeping track of multiple school assignments across different subjects. I wanted a centralized platform that would allow students to view, add, edit, and delete assignments easily, ensuring nothing gets overlooked. My thought process involved designing an intuitive CRUD system, planning both front-end and back-end logic carefully, and integrating it with Node.js and Express for persistent storage. One of the main struggles was handling asynchronous server requests and ensuring real-time updates without breaking the user experience. This project enhanced my understanding of full-stack development, state management, and practical problem-solving in application design.

Habit Tracker Web App
Inspiration & Thought Process

I was inspired to develop the Eco Green Habit Tracker as part of a team initiative to support the Singapore Green Plan 2030 and promote sustainable lifestyles. I wanted to create an intuitive mobile application that encourages users to adopt eco-friendly habits like recycling, saving energy, and using public transport. My thought process focused on linking each habit to logged activities for measurable progress, designing a clean interface, and integrating a backend RESTful web service for data persistence. The biggest challenges were handling CRUD operations efficiently, ensuring smooth API communication, and providing meaningful visual feedback to keep users motivated and engaged.

Tools Used: React Native, RESTful API, Node.js, Express, MongoDB

  • RESTful API Integration
  • Habit-Activity Linking
  • CRUD Operations
  • Time-Based Tracking
  • User-Friendly Mobile UI
  • Data Persistence
